# AutoNetLab Session Database

This folder contains the initial SQLite database design for the AutoNetLab project.

The database is used to store lab session information, topology metadata, injected errors, and validation results. It supports the Session & Results Storage part of the system architecture.

## Files

| File | Description |
|---|---|
| `schema.sql` | Defines the SQLite database tables. |
| `init_db.py` | Creates the local SQLite database and inserts demo data. |
| `autonetlab.db` | Local SQLite database file generated by `init_db.py`. This file is ignored by Git. |

## Tables

### users

Stores basic user/student information.

Main fields:

- `id`
- `username`
- `full_name`
- `role`
- `created_at`

### topologies

Stores topology metadata used by lab sessions.

Main fields:

- `id`
- `topology_code`
- `name`
- `difficulty`
- `description`
- `yaml_path`
- `created_at`

### lab_sessions

Stores created lab sessions.

Main fields:

- `id`
- `session_code`
- `user_id`
- `topology_id`
- `difficulty`
- `status`
- `score`
- `progress`
- `started_at`
- `completed_at`

### injected_errors

Stores configuration errors injected into a lab session.

Main fields:

- `id`
- `session_id`
- `device_name`
- `error_type`
- `description`
- `expected_fix`
- `is_fixed`

### validation_results

Stores validation/scoring results produced after checking a lab session.

Main fields:

- `id`
- `session_id`
- `check_name`
- `status`
- `message`
- `related_topic`
- `score_impact`
- `checked_at`

## Relation with Frontend Mock Data

The frontend currently works with mock JSON files until the real backend API is fully connected.

| Frontend Mock File | Related Database Table |
|---|---|
| `mock_session.json` | `lab_sessions`, `users` |
| `mock_topology.json` | `topologies` |
| `mock_validation_result.json` | `validation_results` |
| `mock_recommendation.json` | Future recommendation module output |
